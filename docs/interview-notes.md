# 面试讲解口径

## 1. 项目一句话介绍

这是一个我基于华盟云门店经营业务逻辑整理的自动化测试 demo，用 Python + pytest 模拟商品、订单、支付、库存、会员、权限和 AI 智能运营助手等核心链路，主要用于练习接口自动化回归和业务断言设计。

## 2. 为什么做这个项目

我之前更多是手工测试、接口测试和回归测试。为了往自动化测试方向提升，我把自己熟悉的业务链路拆成可重复执行的 pytest 用例。这样既能练习 Python 自动化，也能把业务经验沉淀下来。

## 3. 项目重点

我没有把重点放在“搭一个很复杂的框架”，而是先把核心业务场景自动化：

- 商品上下架后多端展示一致
- 下单时金额、优惠、订单状态正确
- 支付回调后订单状态、库存、会员积分同步变化
- 重复支付回调不能重复扣库存
- 不同角色访问敏感报表时有权限控制
- AI 助手回复经营数据时，必须和后台报表数据一致

## 4. 如果面试官问：这是连真实接口吗？

可以这样回答：

目前这个 demo 没有连接真实公司接口，主要是为了避免泄露内部接口和数据，所以使用本地 fake API 模拟业务响应。但用例设计是按照真实业务风险整理的。如果接入测试环境，只需要把 fake API 替换成 requests/httpx 封装的真实接口 client，测试断言和用例组织方式可以保留。

## 5. 如果面试官问：你的自动化水平到哪里？

可以这样回答：

我目前能够使用 Python + pytest 编写基础接口自动化用例，能做 fixture、测试数据、业务断言和简单的回归链路组织。现阶段我更重视业务链路和断言设计，比如金额、状态、库存、权限、数据一致性这些点。后续我希望继续提升接口封装、数据库校验、CI 集成和报告平台能力。

## 6. 如果面试官问：AI Agent 怎么参与测试？

可以这样回答：

AI Agent 更适合做辅助，不适合完全替代测试判断。我会用它根据需求文档、接口字段和历史问题生成测试点初稿，再由我人工筛选和补充断言。比如商品、订单、支付、会员、权限、AI 助手这些模块，AI 可以帮我列边界场景和异常场景，但最终哪些是核心链路、哪些要进入回归，还是要结合业务风险判断。

## 7. 简历可写法

- 基于 pytest 搭建接口自动化测试 demo，围绕商品、订单、支付、库存、会员、权限和 AI 助手等核心链路设计回归用例。
- 通过 fixture 管理测试对象和登录态，使用业务断言校验订单金额、支付状态、库存扣减、会员积分和报表数据一致性。
- 设计 AI 助手测试场景，校验经营数据问答结果与后台报表一致，并覆盖无权限访问和模糊问题兜底。
